Create a reusable TypeScript npm package called `button-carousel`.

Use the same general project/file structure:

```txt

button-carousel/

  src/

    index.ts

    style.ts

  example/

    src/

      main.ts

      style.css

    index.html

    package.json

    vite.config.ts

  package.json

  tsconfig.app.json
  tsconfig.json
  tsconfig.node.json
  tsconfig.test.json

  README.md

```
The tsconfig files should already be set up so dont touch those.

The package should be framework-agnostic and usable from any frontend project with minimal setup.

The package should export a main function:

```ts

import { createButtonCarousel } from "button-carousel";

```

The function should mount an interactive button carousel into a provided container element.

Example API:

```ts

const carousel = createButtonCarousel({

  container: document.querySelector("#carousel")!,

  items: [

    {

      type: "image",

      src: "/assets/image-1.jpg",

      alt: "Example image"

    },

    {

      type: "gif",

      src: "/assets/demo.gif",

      alt: "Example gif"

    },

    {

      type: "video",

      src: "/assets/demo.mp4",

      alt: "Example video"

    }

  ],

  initialIndex: 0

});

```

The returned object should include:

```ts

{

  goTo(index: number): void;

  next(): void;

  previous(): void;

  destroy(): void;

}

```

Requirements:

1. Package behavior

   * Render a row of rounded media buttons.

   * Each button should use the provided image, GIF, or video as the visual fill/background of the button.

   * Images and GIFs should use `<img>`.

   * Videos should use `<video muted loop playsInline autoplay>`.

   * The media should cover the full button using `object-fit: cover`.

   * Clicking a button should make it the active selected button.

   * When a button is active/pressed, the media fill should visually reduce/dim to about 20% opacity.

   * When active/pressed, the shadow should move inward using an inset shadow.

   * Inactive buttons should have an outer shadow.

   * Clicking a different button should animate the selected state shifting from the old active button to the new active button.

   * The component should work with 1 item or many items.

   * The component should not require React, Vue, or any framework.

2. Styling

   * Put all default package styling in `src/style.ts` as an exported template string.

   * Inject the CSS automatically when `createButtonCarousel` is called.

   * Prevent duplicate style injection by checking for an existing style tag ID.

   * Use class names prefixed with `button-carousel-` to avoid conflicts.

   * The visual style should follow the attached PNG reference:

     * rounded pill-like buttons

     * media fills inside each button

     * soft drop shadow on inactive buttons

     * active/pressed button appears pushed inward

     * active/pressed button media is dimmed to about 20%

     * smooth movement/transition when changing the selected button

   * Make the layout responsive.

   * On desktop, the carousel should display horizontally.

   * On smaller screens, it should still fit nicely and allow horizontal overflow scrolling if needed.

3. TypeScript

   * Define clear exported types:

     * `ButtonCarouselItem`

     * `ButtonCarouselOptions`

     * `ButtonCarouselInstance`

   * Support media items shaped like this:

```ts

type ButtonCarouselItem =

  | {

      type: "image";

      src: string;

      alt?: string;

    }

  | {

      type: "gif";

      src: string;

      alt?: string;

    }

  | {

      type: "video";

      src: string;

      alt?: string;

      poster?: string;

    };

```

* Also support “image-like objects” by allowing this structure:

```ts

{

  type: "image";

  src: string | { src: string };

  alt?: string;

}

```

* If `src` is an object with a `src` property, use that inner `src` string.

* Keep the implementation strict and type-safe.

4. Example website

   Build an example Vite website in the `example/` folder that demonstrates all modes.

   The example page should include these sections:

   * “Just Images”

     * A carousel containing multiple image items only.

   * “Single Image”

     * A carousel containing exactly 1 image.

   * “Single Video”

     * A carousel containing exactly 1 video.

   * “Single GIF”

     * A carousel containing exactly 1 GIF.

   * “Mixed Media”

     * A carousel using all 3 media modes: image, GIF, and video.

   * “Mixed Media — 10 Items”

     * A carousel using 10 total items.

     * Include images, GIFs, and videos.

     * This section should clearly demonstrate how the buttons animate and shift selected state when clicking different buttons.

5. Example styling

   * The example page should visually resemble the Frame 1.png reference.

   * Use a clean portfolio/demo layout.

   * Include section titles and short descriptions.

   * Center the demo content.

   * Use a light gray page background.

   * Use rounded cards/sections.

   * The carousel buttons should appear as rounded media thumbnails with shadows.

6. Build setup

   * Use `tsup` for building the package.

   * Generate TypeScript declarations.

   * Make sure `package.json` includes proper fields:

     * `main`

     * `module`

     * `types`

     * `exports`

     * `files`

     * `scripts`

   * The package should build to `dist/`.

   * The example should import the local package source directly through Vite aliasing so development is easy.

7. README

   Write a README that includes:

   * package description

   * installation command

   * basic usage example

   * item type examples for image, GIF, and video

   * explanation of the active/pressed visual state

   * API options

   * returned instance methods

   * note that the package is framework-agnostic

8. Important implementation details

   * Do not use React.

   * Do not use external dependencies unless absolutely necessary.

   * Avoid CSS imports like `?raw`; keep injected styles in `src/style.ts`.

   * Use DOM APIs directly.

   * Keep cleanup safe in `destroy()`.

   * Remove event listeners and DOM elements created by the package when destroyed.

   * Make sure multiple carousels can exist on the same page.

   * Do not hardcode a single global carousel instance.

   * Each carousel instance should manage its own selected index.
